# NekoCafé 项目运行说明

> 本文档用于说明：别人拿到本项目代码后，如何配置环境、启动后端、启动前端，并验证系统是否运行成功。

---

## 1. 项目简介

NekoCafé 是一个猫咪主题餐厅智慧预约平台，采用前后端分离架构：

```text
NekoCafe/
├── backend/    Spring Boot 后端项目
├── frontend/   Vue 3 前端项目
├── docs/       项目文档
├── db/         数据库迁移与备份说明
├── infra/      部署和运维说明
└── tests/      测试说明
```

主要技术栈：

| 部分 | 技术 |
|---|---|
| 后端 | Spring Boot 3、Spring Security、JWT、MyBatis-Plus、Flyway |
| 前端 | Vue 3、Vite、TypeScript、Vue Router、Pinia、Element Plus |
| 数据库 | MySQL 8 |
| 可选服务 | Redis 7 |

---

## 2. 运行前准备

请先安装以下软件：

| 软件 | 建议版本 | 用途 |
|---|---|---|
| JDK | 17 或以上 | 运行 Spring Boot 后端 |
| Maven | 3.9 或以上 | 下载后端依赖、启动后端 |
| Node.js | 20 或以上 | 运行 Vue 前端 |
| npm | 10 或以上 | 下载前端依赖、启动前端 |
| MySQL | 8.0 或以上 | 项目数据库 |
| Git | 任意新版 | 获取源码 |

检查命令：

```bash
java -version
mvn -version
node -v
npm -v
mysql --version
```

## 4. 配置数据库和环境变量

### 4.1 创建 MySQL 数据库

先确保 MySQL 服务已经启动，然后创建数据库：

```sql
CREATE DATABASE nekocafe DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

建议创建单独的数据库账号，不要直接使用 root：

```sql
CREATE USER 'nekocafe_app'@'%' IDENTIFIED BY 'change_me';
GRANT ALL PRIVILEGES ON nekocafe.* TO 'nekocafe_app'@'%';
FLUSH PRIVILEGES;
```

> 如果只是在本机演示，也可以把 `%` 改成 `localhost`。

### 4.2 配置项目 `.env`

在项目根目录 `NekoCafe/` 下复制环境变量示例文件：

```bash
cp .env.example .env
```

然后打开 `.env`，按自己的数据库信息修改：

```text
NEKO_PROFILE=dev

MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=nekocafe
MYSQL_USERNAME=nekocafe_app
MYSQL_PASSWORD=change_me
MYSQL_USE_SSL=false
MYSQL_ALLOW_PUBLIC_KEY_RETRIEVAL=false

JWT_SECRET=change_me_to_at_least_32_chars
JWT_EXPIRES_MINUTES=120

REDIS_HOST=localhost
REDIS_PORT=6379

VITE_API_BASE_URL=http://localhost:8080/api
```

重点说明：

- `MYSQL_HOST`：数据库地址，本地数据库一般填 `localhost`。
- `MYSQL_PORT`：MySQL 端口，默认 `3306`。
- `MYSQL_DATABASE`：数据库名，默认 `nekocafe`。
- `MYSQL_USERNAME` / `MYSQL_PASSWORD`：数据库账号和密码。
- `JWT_SECRET`：JWT 密钥，至少 32 位字符。
- `VITE_API_BASE_URL`：前端访问后端接口的地址，默认 `http://localhost:8080/api`。

> 注意：不要把真实 `.env`、数据库密码、JWT 密钥提交到 Git，也不要截图发到公开群里。

## 5. 启动后端

### 方式一：使用 Makefile 启动，推荐

在 `NekoCafe/` 根目录执行：

```bash
make dev-backend
```

该命令会自动读取根目录 `.env`，然后启动 Spring Boot 后端。

后端默认地址：

```text
http://localhost:8080
```

### 方式二：手动启动

如果电脑没有 `make` 命令，可以手动执行：

```bash
set -a
source .env
set +a
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

如果是在 Windows PowerShell 中，可以使用类似方式手动配置环境变量，或直接在 IDE 的运行配置中填写 `.env` 里的变量。

### 5.1 验证后端是否启动成功

后端启动成功后，浏览器访问：

```text
http://localhost:8080/actuator/health
```

如果看到类似内容，说明后端启动成功：

```json
{"status":"UP"}
```

也可以访问项目健康检查接口：

```text
http://localhost:8080/api/health
```

Swagger 接口文档地址：

```text
http://localhost:8080/swagger-ui/index.html
```

## 6. 启动前端

新开一个终端窗口，进入前端目录：

```bash
cd frontend
```

第一次运行需要安装依赖：

```bash
npm install
```

复制前端环境变量示例文件：

```bash
cp .env.example .env
```

确认 `frontend/.env.local` 中的接口地址为：

```text
VITE_API_BASE_URL=http://localhost:8080/api
```

启动前端：

```bash
npm run dev
```

前端默认地址：

```text
http://localhost:5173
```

浏览器打开该地址即可访问系统。

---

## 7. 推荐启动顺序

完整运行时建议按以下顺序操作：

```text
1. 启动 MySQL
2. 创建 nekocafe 数据库
3. 配置 NekoCafe/.env
4. 启动后端 backend
5. 后端启动成功后，启动前端 frontend
6. 浏览器访问 http://localhost:5173
```

不要先启动前端再启动后端，否则页面可能会因为接口连接失败而显示错误提示。

## 8. 数据库初始化说明

本项目后端集成了 Flyway，启动后端时会自动执行数据库迁移脚本：

```text
backend/src/main/resources/db/migration/V001__init_mysql_schema.sql
backend/src/main/resources/db/migration/V002__seed_basic_data.sql
```

这些脚本会自动创建基础表和初始化角色数据，例如：

```text
CUSTOMER        顾客
STAFF           店员
STORE_MANAGER   店长
HQ_OPERATOR     总部运营
CAT_CARETAKER   猫咪管家
ADMIN           系统管理员
```

如果后端启动时报数据库表相关错误，可以检查：

1. `.env` 中数据库连接信息是否正确。
2. MySQL 是否已经启动。
3. `nekocafe` 数据库是否已经创建。
4. 当前数据库账号是否有建表权限。

## 10. 可选：使用 Docker 启动本地 MySQL

如果没有远程 MySQL，也可以使用项目里的 Docker Compose 启动本地 MySQL：

```bash
docker compose --profile local-mysql up -d mysql-local
```

默认配置：

```text
数据库：nekocafe
账号：nekocafe_app
密码：change_me
端口：3306
```

然后 `.env` 可以配置为：

```text
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=nekocafe
MYSQL_USERNAME=nekocafe_app
MYSQL_PASSWORD=change_me
```

如果需要启动 Redis：

```bash
docker compose up -d redis
```

当前项目 Redis 是可选服务，不启动 Redis 一般不影响基础功能演示。

## 11. 角色页面验证

项目已配置按角色展示页面，不同角色只能看到自己的功能入口。

可验证的页面包括：

| 角色 | 登录后首页 | 可见功能 |
|---|---|---|
| 顾客 `CUSTOMER` | `/stores` | 门店、预约、我的预约、菜单、结算 |
| 店员 `STAFF` | `/staff` | 店员后台、预约签到、订单履约 |
| 猫咪管家 `CAT_CARETAKER` | `/cats` | 猫咪档案 |
| 店长 `STORE_MANAGER` | `/manager` | 店长管理 |
| 总部运营 `HQ_OPERATOR` | `/dashboard` | 数据看板 |
| 管理员 `ADMIN` | `/admin` | 系统管理、数据看板 |

如果某个角色手动访问其他角色页面，例如顾客访问：

```text
http://localhost:5173/admin
```

应该进入：

```text
/403
```

说明权限隔离生效。

## 12. 常见问题

### 12.1 后端启动失败：数据库连接失败

检查：

```text
MYSQL_HOST
MYSQL_PORT
MYSQL_DATABASE
MYSQL_USERNAME
MYSQL_PASSWORD
```

同时确认 MySQL 服务已启动，并且账号有访问 `nekocafe` 数据库的权限。

### 12.2 后端启动失败：JWT_SECRET 太短或配置错误

请把 `.env` 中的 `JWT_SECRET` 改为至少 32 位字符，例如：

```text
JWT_SECRET=nekocafe_dev_secret_change_me_2026
```

不要在正式环境使用示例密钥。

### 12.3 前端页面打不开

检查：

1. 是否已经执行 `npm install`。
2. 是否在 `frontend/` 目录执行 `npm run dev`。
3. 浏览器访问地址是否为 `http://localhost:5173`。

### 12.4 前端接口请求失败

检查：

1. 后端是否已经启动。
2. 后端端口是否为 `8080`。
3. `frontend/.env.local` 中是否配置：

```text
VITE_API_BASE_URL=http://localhost:8080/api
```

### 12.5 登录后看不到其他角色页面

这是正常现象。系统已经做了角色权限隔离：

- 顾客不能看到店员、店长、管理员页面。
- 店员不能看到顾客、店长、管理员页面。
- 其他角色同理。

如果要测试不同角色，需要使用对应角色账号登录。

### 12.6 访问页面显示 403

说明当前账号没有权限访问该页面。解决方式：

1. 检查当前账号角色是否正确。
2. 使用对应角色账号重新登录。
3. 不要删除路由守卫或后端权限注解。
