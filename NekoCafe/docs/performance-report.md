## 3. P95 ≤ 350ms 达标分析

### 达标场景（预期）
- 门店列表：无数据库写入，缓存友好
- 预约时段查询：单表查询，索引优化后可达标
- 看板接口：聚合查询，数据量小时可达标

### 可能不达标场景
- 创建预约：含乐观锁更新，竞争激烈时响应变慢
- 创建订单：多表关联校验 + 优惠券计算
- 沙箱支付：多步骤事务（支付记录 + 积分 + 优惠券状态）
- 登录：BCrypt 密码验证本身耗时较高

### 优化建议（如未达标）
1. 对高频查询字段添加数据库索引
2. 门店列表、菜单等读多写少接口加 Redis 缓存
3. 预约时段查询使用覆盖索引
4. 支付流程考虑异步化积分发放
