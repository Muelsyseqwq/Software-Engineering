## 2. 远程 MySQL 连接方式：SSH 隧道

项目使用远程服务器上的 MySQL。组员本地不要直接连公网 MySQL，而是先开一个 SSH 隧道。

### 2.1 隧道信息

| 项 | 值 |
|---|---|
| 服务器 IP | `118.178.233.223` |
| 本地转发端口 | `3307` |
| 远程 MySQL 地址 | `127.0.0.1` |
| 远程 MySQL 端口 | `3306` |

含义：

```text
本地 localhost:3307  ->  服务器 127.0.0.1:3306
```

后端实际连接本地 `localhost:3307`，SSH 会自动转发到服务器 MySQL。

### 2.2 开启 SSH 隧道

打开 Git Bash，执行下面命令：

```bash
ssh -N -L 3307:127.0.0.1:3306 用户名@118.178.233.223
```

如果服务器用户名是 `root`：

```bash
ssh -N -L 3307:127.0.0.1:3306 root@118.178.233.223
```

如果服务器用户名是 `ubuntu`：

```bash
ssh -N -L 3307:127.0.0.1:3306 ubuntu@118.178.233.223
```

如果用 SSH 密钥登录：

```bash
ssh -i ~/.ssh/你的密钥文件 -N -L 3307:127.0.0.1:3306 用户名@118.178.233.223
```

> 这个命令执行后，终端会停住不动，这是正常现象，表示隧道正在运行。不要关闭这个窗口。

### 2.3 判断 SSH 隧道是否正常

可以再开一个 Git Bash，测试：

```bash
mysql -h 127.0.0.1 -P 3307 -u 数据库用户名 -p
```

如果能输入密码并进入 MySQL，说明隧道正常。

如果电脑没有安装 mysql 命令，也可以先跳过这一步，直接启动后端。如果后端能启动成功，也说明隧道可用。

# 版本一：命令行运行方式

## 3. 拉取代码

打开 Git Bash：

```bash
git clone https://github.com/Muelsyseqwq/Software-Engineering.git
cd Software-Engineering/NekoCafe
```

如果已经拉过仓库：

```bash
cd Software-Engineering
git pull origin main
cd NekoCafe
```

---

## 4. 配置 `.env`

在 `NekoCafe/` 目录下复制环境变量文件：

```bash
cp .env.example .env
```

然后打开 `.env`，按下面格式配置：

```text
NEKO_PROFILE=dev

MYSQL_HOST=localhost
MYSQL_PORT=3307
MYSQL_DATABASE=SE
MYSQL_USERNAME=你的数据库用户名
MYSQL_PASSWORD=你的数据库密码
MYSQL_USE_SSL=false
MYSQL_ALLOW_PUBLIC_KEY_RETRIEVAL=false

JWT_SECRET=nekocafe_dev_secret_change_me_2026
JWT_EXPIRES_MINUTES=120

REDIS_HOST=localhost
REDIS_PORT=6379

VITE_API_BASE_URL=http://localhost:8080/api
```

重点：

```text
MYSQL_HOST=localhost
MYSQL_PORT=3307
```

不要写服务器 IP，因为我们已经通过 SSH 隧道把远程 MySQL 转发到了本地 `3307`。

> `MYSQL_DATABASE` 请以组长给的数据库名为准。如果组长说数据库名是 `SE`，就填 `SE`；如果说是 `nekocafe`，就填 `nekocafe`。

---

## 5. 命令行启动后端

确认 SSH 隧道窗口还开着，然后在新的 Git Bash 中执行：

```bash
cd Software-Engineering/NekoCafe
set -a
source .env
set +a
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

启动成功时，会看到类似：

```text
Tomcat started on port 8080
Started NekoCafeApplication
```

验证后端：

```text
http://127.0.0.1:8080/actuator/health
```

看到下面内容表示成功：

```json
{"status":"UP"}
```

Swagger 地址：

```text
http://127.0.0.1:8080/swagger-ui/index.html
```

---

## 6. 命令行启动前端

再新开一个 Git Bash：

```bash
cd Software-Engineering/NekoCafe/frontend
npm install
cp .env.example .env.local
npm run dev -- --host 127.0.0.1
```

启动成功时，会看到：

```text
Local: http://127.0.0.1:5173/
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

登录页地址：

```text
http://127.0.0.1:5173/login
```

---

## 7. 命令行版本完整启动顺序

每次运行项目，建议按这个顺序：

```text
1. 打开 Git Bash，启动 SSH 隧道
2. 不要关闭 SSH 隧道窗口
3. 新开 Git Bash，启动后端 backend
4. 新开 Git Bash，启动前端 frontend
5. 浏览器访问 http://127.0.0.1:5173/
```

# 版本二：IntelliJ IDEA 运行方式

## 8. IDEA 打开项目

1. 打开 IntelliJ IDEA。
2. 选择 `Open`。
3. 选择项目中的后端目录：

```text
Software-Engineering/NekoCafe/backend
```

4. IDEA 会识别 Maven 项目并下载依赖。
5. 确认 Project SDK 是 JDK 17。

如果没有自动下载依赖，可以打开右侧 Maven 面板，点击刷新按钮。

---

## 9. IDEA 配置后端环境变量

后端入口类：

```text
com.nekocafe.NekoCafeApplication
```

文件位置：

```text
backend/src/main/java/com/nekocafe/NekoCafeApplication.java
```

### 9.1 新建 Spring Boot 运行配置

1. IDEA 右上角点击运行配置下拉框。
2. 选择 `Edit Configurations...`。
3. 点击 `+`。
4. 选择 `Spring Boot`。
5. 填写：

```text
Name: NekoCafe Backend
Main class: com.nekocafe.NekoCafeApplication
Working directory: .../Software-Engineering/NekoCafe/backend
```

### 9.2 配置 Active Profile

在运行配置里设置：

```text
Active profiles: dev
```

如果没有 Active profiles 输入框，可以在 VM options 中填：

```text
-Dspring.profiles.active=dev
```

### 9.3 配置 Environment variables

在 `Environment variables` 中填写以下变量：

```text
NEKO_PROFILE=dev;MYSQL_HOST=localhost;MYSQL_PORT=3307;MYSQL_DATABASE=SE;MYSQL_USERNAME=你的数据库用户名;MYSQL_PASSWORD=你的数据库密码;MYSQL_USE_SSL=false;MYSQL_ALLOW_PUBLIC_KEY_RETRIEVAL=false;JWT_SECRET=nekocafe_dev_secret_change_me_2026;JWT_EXPIRES_MINUTES=120
```

注意：

- Windows / IDEA 里多个环境变量一般用英文分号 `;` 分隔。
- 不要把真实数据库密码截图发群里。
- `MYSQL_DATABASE` 以组长提供的数据库名为准。

## 10. IDEA 运行后端前必须先开 SSH 隧道

IDEA 只负责启动 Java 后端，不负责自动开 SSH 隧道。

所以运行 IDEA 后端前，仍然要先打开 Git Bash，执行：

```bash

```

保持这个 Git Bash 窗口不要关闭。

然后回到 IDEA，点击运行 `NekoCafe Backend`。

启动成功后访问：

```text
http://127.0.0.1:8080/actuator/health
http://127.0.0.1:8080/swagger-ui/index.html
```

# 13. 常见问题排查

## 13.1 后端报 Communications link failure

典型错误：

```text
Communications link failure
Connection refused
```

原因：后端连不上 MySQL。

检查：

1. SSH 隧道窗口是否还开着。
2. 隧道命令本地端口是否是 `3307`。
3. `.env` 或 IDEA 环境变量是否是：

```text
MYSQL_HOST=localhost
MYSQL_PORT=3307
```

4. 数据库用户名、密码、数据库名是否正确。

---

## 13.2 端口 3307 被占用

如果 SSH 隧道启动时报端口被占用，可以换成本地 `13307`：

```bash
ssh -N -L 13307:127.0.0.1:3306 用户名@118.178.233.223
```

然后配置改为：

```text
MYSQL_PORT=13307
```

---

## 13.3 后端 8080 被占用

如果后端启动时报 8080 被占用，可以先关闭占用 8080 的程序。

Windows 可以查看占用：

```bash
netstat -ano | findstr :8080
```

然后在任务管理器中结束对应 PID。

---

## 13.4 前端 5173 被占用

Vite 一般会自动换端口，比如 `5174`。以终端输出为准：

```text
Local: http://127.0.0.1:5174/
```

---

## 13.5 登录页在哪里

登录页地址：

```text
http://127.0.0.1:5173/login
```

如果访问 `/login` 自动跳走，说明浏览器之前登录过。可以清空浏览器 localStorage：

```js
localStorage.clear()
location.href = '/login'
```

---

## 13.6 前端接口请求失败

检查：

1. 后端是否启动成功。
2. 后端是否能访问：

```text
http://127.0.0.1:8080/actuator/health
```

3. `frontend/.env.local` 是否是：

```text
VITE_API_BASE_URL=http://localhost:8080/api
```
