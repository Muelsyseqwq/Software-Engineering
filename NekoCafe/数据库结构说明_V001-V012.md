# NekoCafé 数据库结构说明（V001-V012）

本文档根据 `backend/src/main/resources/db/migration/` 下的 Flyway 迁移脚本整理，说明当前新数据库执行 `V001` 到 `V012` 后包含的表、字段、主要约束和演示数据。

> 数据库类型：MySQL 8  
> 字符集：`utf8mb4`  
> 迁移工具：Flyway  
> 迁移目录：`NekoCafe/backend/src/main/resources/db/migration/`

---

## 1. 迁移脚本总览

| 版本 | 文件 | 作用 |
|---|---|---|
| V001 | `V001__init_mysql_schema.sql` | 创建基础业务表：用户、角色、门店、桌位、预约、菜单、订单、支付、看板统计 |
| V002 | `V002__seed_basic_data.sql` | 初始化角色、1 号门店、桌位、菜品分类和基础菜品 |
| V003 | `V003__seed_customer_demo.sql` | 补充顾客流程演示菜品和未来 7 天预约时段 |
| V004 | `V004__seed_demo_role_users.sql` | 初始化各角色演示账号 |
| V005 | `V005__add_cat_and_activity_tables.sql` | 新增猫咪、猫咪排班、预约猫咪关联、活动、活动门店关系表 |
| V006 | `V006__add_review_refund_member_preference_tables.sql` | 新增评价、退款、积分流水、用户偏好表 |
| V007 | `V007__add_store_staff_shift_and_order_fields.sql` | 新增门店人员关系、员工排班、桌位状态日志，并扩展订单和预约字段 |
| V008 | `V008__normalize_business_status_and_store_location.sql` | 扩展门店位置/面积字段，统一部分业务状态值 |
| V009 | `V009__add_manager_leave_and_price_history.sql` | 新增请假/放假表和菜品改价历史表 |
| V010 | `V010__seed_unified_demo_data.sql` | 初始化猫咪、活动、门店绑定、排班、偏好、积分等统一演示数据 |
| V011 | `V011__merge_admin_into_hq_operator_role.sql` | 合并总部运营展示语义，保留 ADMIN 兼容角色 |
| V012 | `V012__seed_system_test_data.sql` | 初始化多门店、多店员、多订单、多活动等系统测试数据 |

# 3. 用户权限相关表

## 3.1 `user` 用户账号表

用途：保存所有系统登录用户，包括顾客、店员、店长、猫咪管家、总部运营和兼容管理员。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT PK AUTO_INCREMENT | 用户 ID |
| `username` | VARCHAR(64) NOT NULL | 用户名，唯一 |
| `password_hash` | VARCHAR(255) NOT NULL | BCrypt 密码哈希 |
| `nickname` | VARCHAR(64) NOT NULL | 昵称 |
| `phone` | VARCHAR(32) NULL | 手机号，唯一，可空 |
| `email` | VARCHAR(128) NULL | 邮箱，唯一，可空 |
| `status` | VARCHAR(32) NOT NULL DEFAULT `ACTIVE` | 账号状态 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |
| `deleted` | TINYINT DEFAULT 0 | 逻辑删除标记 |
| `version` | INT DEFAULT 0 | 乐观锁版本 |

主要约束/索引：

- `uk_user_username (username)`
- `uk_user_phone (phone)`
- `uk_user_email (email)`

常见状态：

- `ACTIVE`：可用
- 后续可扩展：`DISABLED`、`LOCKED`

---

## 3.2 `role` 角色表

用途：保存系统角色定义。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT PK AUTO_INCREMENT | 角色 ID |
| `code` | VARCHAR(64) NOT NULL | 角色编码，唯一 |
| `name` | VARCHAR(64) NOT NULL | 角色名称 |
| `description` | VARCHAR(255) NULL | 角色说明 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

主要约束：

- `uk_role_code (code)`

当前角色：

| code | name | 说明 |
|---|---|---|
| `CUSTOMER` | 顾客 | 顾客端用户 |
| `STAFF` | 店员 | 门店一线工作人员 |
| `STORE_MANAGER` | 店长 | 门店管理者 |
| `HQ_OPERATOR` | 总部运营 | 跨门店数据、活动运营、用户角色和平台配置管理 |
| `CAT_CARETAKER` | 猫咪管家 | 猫咪档案和健康状态维护 |
| `ADMIN` | 总部运营兼容角色 | 历史系统管理员兼容角色 |

说明：`V011` 将总部运营与后台管理展示合并，但保留 `ADMIN` 兼容旧数据。

# 15. 协作注意事项

1. `V001` 到 `V012` 都是递进迁移，不是覆盖关系。
2. 已执行过的迁移文件不要修改，否则会触发 Flyway checksum mismatch。
3. 后续如果需要新增表、字段或演示数据，继续新增：

```text
V013__xxx.sql
V014__xxx.sql
```

4. 不要把真实数据库密码、JWT 密钥、云服务器地址等敏感信息写入迁移脚本、README、截图或日志。
5. 建议组员统一连接已经执行到最新版本的数据库，例如 `nekocafe_unified`。
