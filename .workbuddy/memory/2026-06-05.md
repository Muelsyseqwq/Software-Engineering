# 2026-06-05 工作日志

## 店员订单履约：支持查看已处理历史订单

### 需求
店员需要能看到被自己处理过的所有订单信息，而不是处理完（COMPLETED）后就从列表消失。

### 后端修改
1. **数据库迁移**: 新增 `V005__add_handler_id_to_food_order.sql`，给 `food_order` 表增加 `handler_id` 字段并建索引
2. **FoodOrder 实体**: 增加 `handlerId` 字段及 getter/setter
3. **StaffService**:
   - `startOrder(Long id, Long staffId)`: 开始制作时记录当前店员ID到 `handlerId`
   - 新增 `handledOrders(Long staffId)`: 查询 `handler_id = staffId` 的所有订单（包括制作中和已完成的）
4. **StaffController**:
   - `startOrder` 增加 `@AuthenticationPrincipal AuthPrincipal principal` 参数，传入当前用户ID
   - 新增 `GET /api/staff/orders/handled` 接口，返回当前店员处理过的所有订单

### 前端修改
1. **staff.ts**: 新增 `fetchHandledOrders()` API 调用
2. **OrderFulfillmentView.vue**:
   - 使用 `el-tabs` 分为"待处理订单"和"已处理订单"两个标签页
   - "待处理"保持原有功能（开始制作、完成）
   - "已处理"只读展示，显示该店员点击过"开始制作"的所有历史订单
   - 切换标签页时自动加载对应数据
