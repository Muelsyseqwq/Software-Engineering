# 2026-06-08 工作日志

## 数据库变更影响评估与店员角色功能增强

### 数据库变更影响评估
根据 `数据库结构说明_V001-V012.md`，对比现有代码发现以下受影响点：

1. **Reservation 状态值变更（V008）**：旧值 `PENDING_PAYMENT/PAID/CONFIRMED` → 新值 `RESERVED/CHECKED_IN/COMPLETED/CANCELLED/NO_SHOW`
2. **FoodOrder 状态值变更（V008）**：旧值 `COOKING` → 新值 `PREPARING`；新增 `REFUNDING/REFUNDED`
3. **Reservation 表新增字段（V007）**：`checked_out_at`, `completed_at`, `cancelled_at`
4. **FoodOrder 表新增字段（V007）**：`table_id`, `paid_at`, `completed_at`, `cancelled_at`, `refund_status`
5. **Cat 表新增字段（V005）**：`store_id`（Cat.java 实体原先缺少此字段，已补充）

### 已修复的兼容性问题
- `Reservation.java`：增加 `checkedOutAt`, `completedAt`, `cancelledAt`
- `FoodOrder.java`：增加 `tableId`, `paidAt`, `completedAt`, `cancelledAt`, `refundStatus`
- `Cat.java`：增加 `storeId`（属于猫咪管家模块，但数据库变更导致店员查询猫咪功能依赖此字段）
- `StaffService`：状态翻译映射全部按最新字典更新；`startOrder` 状态改为 `PREPARING`；查询条件同步更新

### 新增功能

**① 状态显示中文**
- 预约状态：`RESERVED→已预约`, `CHECKED_IN→已签到`, `COMPLETED→已完成`, `CANCELLED→已取消`, `NO_SHOW→爽约`
- 订单状态：`CREATED→待支付`, `PAID→已支付/待制作`, `PREPARING→制作中`, `COMPLETED→已完成`, `CANCELLED→已取消`, `REFUNDING→退款中`, `REFUNDED→已退款`

**② 预约备注和时段**
- `StaffReservationRow` 新增 `remark`（备注）和 `timeSlot`（预约时段，如 "14:00 - 16:00"）
- `StaffService.todayReservations()` 关联 `reservation_slot` 表查询时段信息
- 前端 `ReservationCheckInView.vue` 新增"预约时段"和"备注"列

**③ 桌位和猫咪状态查询与修改**
- 新建 `DiningTableStatusLog.java` 实体和 Mapper
- 新建 `UserStoreRole.java` 实体和 Mapper（用于获取店员所属门店）
- 新增接口：`GET /api/staff/tables`（支持按状态筛选）、`GET /api/staff/cats`（支持按状态筛选）、`PUT /api/staff/tables/{id}/status`
- 修改桌位状态时自动写入 `dining_table_status_log` 日志表
- 新建前端页面 `TableCatStatusView.vue`，支持双标签页展示、状态筛选、弹窗修改桌位状态

**④ 订单关联桌位信息**
- `StaffOrderRow` 新增 `tableNo` 字段
- `StaffService` 中订单查询关联 `dining_table` 表获取桌号
- 前端 `OrderFulfillmentView.vue` 和 `StaffHomeView.vue` 新增"桌号"列
